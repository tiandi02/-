================================================================================
            HwMonitor64 自动关闭监控程序 - 详细说明文档
================================================================================

【程序概述】
本程序的功能：当 HwMonitor64.exe 启动时，自动检测并终止该进程。
运行方式：后台每隔 2 秒扫描一次系统进程列表，发现目标进程则立即终止。


================================================================================
一、对 CPU 的影响
================================================================================

1. 空闲时的 CPU 占用
   - 每次扫描耗时约 1~5 毫秒（取决于系统进程数量）
   - 每 2 秒扫描一次，CPU 占用率约为 0.05%~0.2%
   - 在任务管理器中几乎不可察觉，可视为零负担

2. 扫描时的瞬间峰值
   - psutil.process_iter() 调用会短暂占用 CPU
   - 峰值持续时间极短（毫秒级），不会造成卡顿
   - 即使在低端 CPU 上也不会感知到任何影响

3. 长时间运行的累积影响
   - 无累积效应，24小时运行的 CPU 总占用与刚启动时一致
   - 不会导致 CPU 温度上升
   - 不影响其他程序的运行性能


================================================================================
二、对 GPU 的影响
================================================================================

1. GPU 占用
   - 本程序不使用任何图形渲染、3D 计算、视频编解码功能
   - 对 GPU 的占用为 0%
   - 不会导致 GPU 温度上升
   - 不影响游戏或图形应用的帧率

2. 显存占用
   - 本程序不使用 GPU 显存
   - 显存占用为 0 MB


================================================================================
三、对内存（RAM）的影响
================================================================================

1. 内存占用
   - Python 解释器本身约占用 15~25 MB
   - psutil 库约占用 2~5 MB
   - 总计约 20~30 MB
   - 对现代电脑（通常 8GB 以上内存）影响可忽略

2. 内存泄漏
   - 不存在内存泄漏问题
   - 长时间运行内存占用保持稳定


================================================================================
四、对磁盘的影响
================================================================================

1. 磁盘读写
   - 仅在终止进程时写入日志文件（每次约几十字节）
   - 无持续的磁盘读写操作
   - 不会影响 SSD 寿命

2. 日志文件大小
   - 每条日志约 80 字节
   - 即使每小时终止 30 次，一天也仅约 57 KB
   - 如需清理，直接删除 kill_log.txt 即可（程序会自动重建）

   - 不上传或下载任何数据
   - 对网络带宽占用为 0

================================================================================
七、如何关闭本监控程序
================================================================================

根据你使用的启动方式，关闭方法如下：

------------------------------------------------------------------------
方式一：通过「启动监控.bat」启动的（有黑色命令行窗口）
------------------------------------------------------------------------
  方法 1：在命令行窗口中按 Ctrl + C 即可停止
  方法 2：直接关闭命令行窗口（点击右上角 X 按钮）
  方法 3：在任务管理器中找到 python.exe，右键 → 结束任务

------------------------------------------------------------------------
方式二：通过「kill_hwmonitor.pyw」启动的（无窗口后台运行）
------------------------------------------------------------------------
  方法 1（推荐）：
    打开任务管理器 → 详细信息选项卡 → 找到 pythonw.exe → 右键 → 结束任务

  方法 2（命令行）：
    打开命令提示符或 PowerShell，输入以下命令：
    taskkill /f /im pythonw.exe
    （注意：这会关闭所有 pythonw.exe 进程）

  方法 3（仅关闭本程序，不影响其他 pythonw.exe）：
    打开命令提示符或 PowerShell，输入以下命令：
    wmic process where "commandline like 'kill_hwmonitor'" call terminate

------------------------------------------------------------------------
方式三：开机自启动（计划任务方式）
------------------------------------------------------------------------
  步骤 1：先按上述方式关闭正在运行的监控程序
  步骤 2：删除计划任务，打开命令提示符（管理员），输入：
          schtasks /delete /tn "KillHwMonitor64" /f
  步骤 3：提示"成功"即表示已取消开机自启


================================================================================
八、常见问题
================================================================================

Q1：怎么确认监控程序是否正在运行？
A1：打开任务管理器 → 详细信息选项卡
    - 看到 python.exe → 通过「启动监控.bat」启动的版本正在运行
    - 看到 pythonw.exe → 后台静默版本正在运行
    - 都没有 → 监控程序未运行

Q2：关闭后 HwMonitor64.exe 还会被自动关闭吗？
A2：不会。监控程序关闭后不再有任何监控功能，HwMonitor64.exe 可正常运行。

Q3：可以修改检查间隔吗？
A3：可以。用文本编辑器打开 kill_hwmonitor.py，找到：
    CHECK_INTERVAL = 2
    将 2 改为你想要的秒数（如 5 表示每 5 秒检查一次）。
    间隔越大，CPU 占用越低，但响应速度越慢。

Q4：日志文件太大怎么办？
A4：直接删除 D:\Claude code\程序\kill_log.txt，程序下次写入时会自动重建。

Q5：为什么有时 HwMonitor64.exe 闪了一下才被关闭？
A5：因为程序每 2 秒检查一次，HwMonitor64.exe 可能在两次检查之间短暂运行。
    缩小 CHECK_INTERVAL 的值可以缩短这个窗口期（最小建议 1 秒）。
