================================================================================
        将 HwMonitor64 自动关闭监控程序开源到 GitHub - 详细步骤指南
================================================================================


================================================================================
第一步：注册 GitHub 账号（已有账号可跳过）
================================================================================

  1. 打开浏览器，访问 https://github.com
  2. 点击右上角 "Sign up"
  3. 输入邮箱地址 → 设置密码 → 设置用户名 → 完成验证
  4. 在邮箱中查收验证邮件并确认


================================================================================
第二步：安装 Git（已安装可跳过）
================================================================================

  1. 访问 https://git-scm.com/download/win
  2. 下载 Windows 版 Git 安装包
  3. 运行安装程序，一路 Next 使用默认选项即可
  4. 安装完成后，打开命令提示符验证：
     git --version
     能显示版本号即安装成功


================================================================================
第三步：配置 Git 用户信息
================================================================================

  打开命令提示符，输入以下命令（替换为你自己的信息）：

  git config --global user.name "你的GitHub用户名"
  git config --global user.email "你的GitHub注册邮箱"

  验证配置：
  git config --global --list


================================================================================
第四步：创建 .gitignore 文件
================================================================================

  在项目目录 "D:\Claude code\程序" 下创建 .gitignore 文件，
  排除不需要上传的文件：

  --------------------------------------------------
  # 日志文件
  kill_log.txt

  # Python 缓存
  __pycache__/
  *.pyc

  # 系统文件
  Thumbs.db
  Desktop.ini
  --------------------------------------------------

  说明：kill_log.txt 是运行时产生的日志，不应上传到仓库；
       程序说明.txt 和 开源发布指南.txt 是文档，可以选择保留或排除。


================================================================================
第五步：编写 README.md（项目首页说明）
================================================================================

  在项目目录下创建 README.md 文件，这是 GitHub 仓库的首页展示内容。
  建议包含以下部分：

  - 项目名称和简介
  - 功能说明
  - 使用方法
  - 运行环境要求
  - 关闭程序的方法
  - 开源许可证
  - 贡献指南

  具体内容见下方第八步的 README.md 模板。


================================================================================
第六步：选择开源许可证
================================================================================

  开源必须选择一个许可证，否则他人无法合法使用你的代码。

  常见许可证对比：

  ┌────────────┬──────────┬──────────┬──────────┬──────────────────────┐
  │  许可证     │ 可商用    │ 可修改    │ 需开源    │  适合场景             │
  ├────────────┼──────────┼──────────┼──────────┤──────────────────────┤
  │  MIT       │  可以     │  可以     │  不需要   │  最自由，推荐新手使用  │
  │  Apache 2.0│  可以     │  可以     │  不需要   │  含专利保护，更严谨    │
  │  GPL 3.0   │  可以     │  可以     │  需要     │  衍生作品必须开源      │
  └────────────┴──────────┴──────────┴──────────┴──────────────────────┘

  推荐：MIT License（最简单宽松，适合此类小工具）

  创建方法：在项目目录下创建 LICENSE 文件，内容见下方第八步。


================================================================================
第七步：在 GitHub 上创建仓库
================================================================================

  1. 登录 GitHub
  2. 点击右上角 "+" → "New repository"
  3. 填写信息：
     - Repository name:  kill-hwmonitor
       （建议用英文短横线分隔，不用中文和空格）
     - Description:  自动监控并关闭 HwMonitor64 的小工具
     - Public（开源选这个）/ Private（私有）
     - 不要勾选 "Add a README file"（我们本地已有）
     - 不要勾选 ".gitignore"（我们本地已有）
     - License 选 "None"（我们本地已有）
  4. 点击 "Create repository"


================================================================================
第八步：在本地初始化 Git 仓库并添加文件
================================================================================

  打开命令提示符，依次执行以下命令：

  --- 进入项目目录 ---
  cd /d "D:\Claude code\程序"

  --- 初始化 Git 仓库 ---
  git init

  --- 添加所有文件到暂存区 ---
  git add .

  --- 查看将要提交的文件 ---
  git status
  （确认文件列表正确，.gitignore 中的文件不应出现）

  --- 提交 ---
  git commit -m "初始提交：HwMonitor64 自动关闭监控程序"


================================================================================
第九步：关联远程仓库并推送
================================================================================

  在 GitHub 创建仓库后，页面会显示远程地址，格式如：
  https://github.com/你的用户名/kill-hwmonitor.git

  在命令提示符中执行：

  --- 关联远程仓库 ---
  git remote add origin https://github.com/你的用户名/kill-hwmonitor.git

  --- 推送到 GitHub ---
  git branch -M main
  git push -u origin main

  首次推送会弹出 GitHub 登录窗口，按提示登录即可。


================================================================================
第十步：验证发布结果
================================================================================

  1. 打开浏览器，访问你的仓库页面：
     https://github.com/你的用户名/kill-hwmonitor

  2. 确认以下内容：
     - README.md 正常显示在首页
     - 所有源代码文件都在
     - LICENSE 文件存在
     - .gitignore 中的文件（kill_log.txt 等）未上传

第十一步：后续更新代码
================================================================================

--------------------------------------------------切割线（不含本行）--------------------------------------------------
# Kill HwMonitor64

自动监控并关闭 HwMonitor64.exe 的小工具。当目标进程启动时，本程序会自动检测并终止它。

## 功能

- 每 2 秒自动扫描系统进程
- 检测到 HwMonitor64.exe 运行时自动终止
- 支持后台静默运行（无窗口）
- 支持设置开机自启动
- 自动记录操作日志

## 运行环境

- Windows 10 / 11
- Python 3.8+
- psutil 库（`pip install psutil`）

## 使用方法

### 手动启动（带窗口）

双击 `启动监控.bat`，在弹出的窗口中按 Ctrl+C 停止。

### 后台静默运行

双击 `kill_hwmonitor.pyw`，无窗口运行。通过任务管理器结束 pythonw.exe 来停止。

### 开机自启动

右键以管理员身份运行 `安装开机自启.bat`。

卸载自启动：
```
schtasks /delete /tn "KillHwMonitor64" /f
```

## 配置

修改 `kill_hwmonitor.py` 中的以下参数：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `CHECK_INTERVAL` | 2 | 检查间隔（秒） |
| `TARGET_PROCESS` | HwMonitor64.exe | 目标进程名 |

## 性能影响

| 资源 | 占用 |
|------|------|
| CPU | 0.05%~0.2% |
| GPU | 0% |
| 内存 | 20~30 MB |
